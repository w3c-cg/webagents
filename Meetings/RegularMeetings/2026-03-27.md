Andrei Ciortea: The previous meeting focused on agent-environment interaction, which is one of the main sections of the interoperability report. On the same day, we presented our CG to the Web \& AI Interest Group.

Antoine Zimmermann (AZ): Interest Group meetings are open for anyone to join. Presentation of a scenario: [https://github.com/w3c-cg/webagents/blob/illuminate-room-scenario/Meetings/RegularMeetings/hmas-scenario.md](https://github.com/w3c-cg/webagents/blob/illuminate-room-scenario/Meetings/RegularMeetings/hmas-scenario.md)

The agent listens to signals to find light sources to brighten the room of its owner. Each signal transmit a single URI. The agent dereferences the perceives URIs to RDF files.

The agent is expecting to find light sources as described in an ontology.

The agent needs a strategy to explore linked data because the information it seeks may not be directly in the perceived RDF files but from a another RDF file it can reach by following links. Using reasoning, the agent may deduce that a device, whose RDF description is discovered by the agent, is a light source from the RDF information.

Jérémy Lemée: refers to this article: [https://dl.acm.org/doi/pdf/10.1145/3603163.3609077](https://dl.acm.org/doi/pdf/10.1145/3603163.3609077) for LLM being used to guide agents in hypermedia environments. Such a similar system could be applied in the proposed scenario.

Andrei Olaru: question on how the traversal is performed.

AZ: The hmas:hosts relation can be used to discover new resources. 

Gustavo Nardin: The traversal is made through links in the ontology to avoid exploring all links most of them probably not relevant.  

AZ: Generic strategy for exploring linked data is difficult. Most URIs are not worth following (e.g., property URIs). However, every link is potentially relevant. Heuristics are important. 

Andrei Ciortea: It is a great scenario because it is simple but also raises many questions. The scenario could be added to the interoperability report because it features many architectural properties. hmas:hosts is properly not the best link to follow. A room should be mapped to a workspace rather than a platform. Language agents could help because they are good at navigating the Web. 

Alexandru Sorici made experiments on smart homes described as hMAS environments. One highlight: exploration on linked data helps LLM agents. Action affordances are very generic. rdfs:comment could be used to provide a natural language description of what the affordance is actually used for. We can consider environments where the breadth of a graph to crawl is bounded or we can consider environments where the breadth is not bounded. In the latter case, the agent needs to know when to stop exploration. Is the ontology there at the beginning or is it created by an LLM agent?

AZ: Formal information is there to provide guarantees to the agent if the ontology is used properly. However, hybrid approaches are possible. The manufacturer of a product could align their own concepts with a standardized taxonomy. The scenario has only one agent but could be extended to a scenario with multiple agents. An software agent (e.g., an agent running on a mobile phone) could ask a physical agent or a human to turn on a physical switch. 

Andrei Ciortea: Search engines are becoming more and more important for agents. A search engine could be used to explore the environment. XML can be used for prompts because structured information helps. In the physical world, structure is very important. In some areas (e.g., Web of Things), structured information is very useful even for LLM agents. 

AZ: dco is a simple language to describe connected objects.  A description initially targetted to hight school students and a broad audience was made at: [https://www.emse.fr/~zimmermann/Teaching/KRR/krr101.pdf](https://www.emse.fr/~zimmermann/Teaching/KRR/krr101.pdf), with a description of a concrete example at [https://www.emse.fr/~zimmermann/LiveInLab-en/](https://www.emse.fr/~zimmermann/LiveInLab-en/)
